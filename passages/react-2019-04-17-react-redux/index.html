UI组件

react-redux将所有组件分成两类：UI组件和容器组件

UI组件负责UI呈现； 容器组件负责管理数据和逻辑

UI 组件特征：

  1. 只负责UI呈现，不带有任何业务逻辑

  2. 没有状态（不使用this.state变量）

  3. 所有数据都由this.props提供

  4. 不使用任何redux 的api

  5. 因没有状态，UI组件又称“纯组件”，仅由参数props决定其值

容器组件：

  1. 负责管理数据和业务逻辑，不负责UI呈现
  2. 带有内部状态
  3. 使用redux的api

当一个组件既有UI又有业务逻辑，则拆分为：外面是一个容器组件，里面包了一个UI组件；前者负责与外部的通信，将数据传给后者，由后者渲染出视图。

React-redux规定，所有UI组件都由用户提供，容器组件则由react-redux自动生成（用户仅负责视觉层，状态管理全交给react-redux）；

connect（）
